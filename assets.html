How the repository files connect

This page gives a quick visual map of the project. It shows which files are entry points, which are reusable UI sections, where styling comes from, and where static assets are served.

Page entry points Reusable components Global styling Static files

Page layer

pages/index.tsx
  • →Imports UI sections like Hero, FAQ, Examples.
  • →Calls GitHub API in getStaticProps to fetch contributors.
  • →Passes fetched data into ExamplesSection.

Component layer

components/*.tsx
  • →Each component renders one website section.
  • →Composed together by pages/index.tsx.
  • →Can reference static logos/images in public/.

Styling layer

styles/globals.css
  • →Loaded via pages/_app.tsx.
  • →Provides global styles and utility-friendly base rules.
  • →Shared by all pages and components.

Static asset layer

public/...
  • →Served directly by Next.js from root URL paths.
  • →Used for favicons, social image, and tool logos.
  • →No import build step needed for basic file references.
pages/_app.tsx ───────► styles/globals.css
       │
       └──────────────► pages/index.tsx ───────► components/*
                                      │
                                      ├────────► public/logos/*
                                      └────────► GitHub API (contributors)
